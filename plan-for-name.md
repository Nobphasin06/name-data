# แผนการพัฒนาแอปพลิเคชัน Name Data

## 1. ภาพรวมของระบบ (System Overview)
แอปพลิเคชันนี้ถูกออกแบบมาเพื่อจัดการข้อมูล (Name Data) โดยเน้นไปที่การทำงานพื้นฐาน (CRUD Operations) ได้แก่ การสร้าง, การอ่าน, การแก้ไข และการลบข้อมูล พร้อมทั้งมี **ระบบยืนยันตัวตน (Authentication) และการสมัครสมาชิก (Registration) โดยใช้ JWT (JSON Web Token)** เพื่อรักษาความปลอดภัยของข้อมูลและจำกัดสิทธิ์การเข้าถึง

## 2. ขอบเขตของระบบ (Scope of Work)
### ฟีเจอร์หลัก (Core Features - CRUD)
* **C - Create (เพิ่มข้อมูล):** ผู้ใช้สามารถเพิ่มข้อมูลใหม่ลงในระบบได้
* **R - Read (อ่าน/แสดงผลข้อมูล):** ผู้ใช้สามารถเรียกดูรายการข้อมูลทั้งหมดที่บันทึกไว้ได้ และสามารถดูรายละเอียดของแต่ละรายการได้
* **U - Update (แก้ไขข้อมูล):** ผู้ใช้สามารถปรับปรุงและแก้ไขข้อมูลที่มีอยู่แล้วในระบบได้
* **D - Delete (ลบข้อมูล):** ผู้ใช้สามารถลบข้อมูลที่ไม่ต้องการออกจากระบบได้

### ฟีเจอร์การยืนยันตัวตน (Authentication - JWT)
* **Register (สมัครสมาชิก):** ผู้ใช้สามารถสร้างบัญชีใหม่เพื่อเข้าใช้งานระบบได้
* **Login (เข้าสู่ระบบ):** ผู้ใช้สามารถเข้าสู่ระบบด้วยอีเมลและรหัสผ่าน โดยระบบจะออก Access Token และ Refresh Token ให้
* **Profile (ข้อมูลส่วนตัว):** ผู้ใช้สามารถดูและแก้ไขข้อมูลส่วนตัวของตนเองได้
* **Logout (ออกจากระบบ):** ผู้ใช้สามารถออกจากระบบและยกเลิกโทเคนได้อย่างปลอดภัย
## 3. โครงสร้างข้อมูลเบื้องต้น (Database Structure)
ตารางสำหรับเก็บข้อมูลหลัก (ตัวอย่างเช่นตาราง `records`)
* `id` (Primary Key): รหัสอ้างอิงของข้อมูล (เช่น UUID หรือ Auto-increment Integer)
* `name` (String): ชื่อข้อมูล หรือ ชื่อผู้ติดต่อ
* `description` (Text): รายละเอียดเพิ่มเติม
* `created_at` (Timestamp): วันเวลาที่สร้างข้อมูล
* `updated_at` (Timestamp): วันเวลาที่มีการแก้ไขข้อมูลล่าสุด

ตาราง `users` (ข้อมูลผู้ใช้งานสำหรับการล็อกอิน)
* `id` (Primary Key): รหัสผู้ใช้งาน
* `email` (String, Unique): อีเมลที่ใช้ในการเข้าสู่ระบบ
* `password_hash` (String): รหัสผ่านที่ถูกเข้ารหัสด้วย bcrypt
* `refresh_token` (String): โทเคนสำหรับขอ Access Token ใหม่
* `is_active` (Boolean): สถานะการใช้งานบัญชี

ตาราง `profiles` (ข้อมูลส่วนตัวของผู้ใช้งาน)
* `id` (Primary Key): รหัสโปรไฟล์
* `user_id` (Foreign Key): รหัสอ้างอิงไปยังตาราง users
* `display_name` (String): ชื่อสำหรับแสดงผล
* `avatar_url` (String): รูปโปรไฟล์
* `bio` (Text): ประวัติย่อ
* `phone` (String): เบอร์โทรศัพท์ส่วนตัว

*(หมายเหตุ: โครงสร้างฟิลด์สามารถปรับเปลี่ยนได้ตามลักษณะข้อมูลจริงที่ต้องการจัดเก็บ)*

## 4. โครงสร้างหน้าจอการใช้งาน (User Interface - UI)
* **หน้าหลัก (List View):**
    * แสดงรายการข้อมูลทั้งหมดในรูปแบบตาราง หรือการ์ด (Card)
    * มีปุ่มสำหรับ "เพิ่มข้อมูลใหม่", "โปรไฟล์ส่วนตัว" และ "ออกจากระบบ"
    * แต่ละรายการจะมีปุ่ม Action สำหรับ "แก้ไข" และ "ลบ"
* **หน้ายืนยันตัวตน (Authentication Views):**
    * **หน้าล็อกอิน (Login):** สำหรับกรอกอีเมลและรหัสผ่านเพื่อเข้าใช้งาน
    * **หน้าสมัครสมาชิก (Register):** สำหรับสร้างบัญชีผู้ใช้ใหม่
* **หน้าโปรไฟล์ (Profile View):**
    * แสดงและแก้ไขข้อมูลส่วนตัว เช่น ชื่อแสดงผล, รูปโปรไฟล์ และประวัติย่อ
* **หน้าฟอร์ม (Form View - สร้าง/แก้ไข):**
    * แบบฟอร์มสำหรับกรอกข้อมูล (ชื่อ, รายละเอียด ฯลฯ)
    * มีปุ่ม "บันทึก" (Save) และปุ่ม "ยกเลิก" (Cancel)
    * ใช้ฟอร์มเดียวกันสำหรับทั้งการเพิ่มข้อมูลใหม่และการแก้ไขข้อมูลเดิม (โดยเช็คจาก `id`)

## 5. สถาปัตยกรรมระบบและเทคโนโลยี (Architecture & Tech Stack)
* **Frontend:** HTML/CSS/JS (Vanilla) ร่วมกับ SweetAlert2
* **Backend API:** Node.js (Express)
* **Database:** SQLite
* **Authentication:** JWT (JSON Web Token) และ bcryptjs
* **CI/CD & Deployment:** Docker และ GitHub Actions สำหรับการ Push ขึ้น Docker Hub อัตโนมัติ
## 6. ขั้นตอนการพัฒนา (Development Steps)
1. **Database Setup:** ออกแบบและสร้างตารางฐานข้อมูลสำหรับเก็บข้อมูล CRUD
2. **API Development:** สร้าง RESTful API สำหรับการทำงานทั้ง 4 รูปแบบ (GET, POST, PUT, DELETE)
3. **UI Development:** สร้างหน้าจอส่วนต่อประสานกับผู้ใช้งาน (Frontend)
4. **Integration & Testing:** เชื่อมต่อ Frontend เข้ากับ Backend API และทดสอบการทำงาน CRUD ทั้งหมดให้ครบถ้วน

## 7. ระบบยืนยันตัวตน (Authentication)
เพื่อความปลอดภัยและการจำกัดสิทธิ์การเข้าถึงข้อมูล จะมีการเพิ่มระบบล็อกอินและสมัครสมาชิก:
* **Database (`users`):** สร้างตารางสำหรับเก็บข้อมูลผู้ใช้งาน (`id`, `username`, `password_hash`)
* **API (Auth):** สร้าง API `/api/register` สำหรับสมัครสมาชิก และ `/api/login` สำหรับล็อกอินเพื่อออก JWT (JSON Web Token) สำหรับใช้ยืนยันตัวตน
* **UI (Auth Forms):** เพิ่มหน้าสำหรับ สมัครสมาชิก (Sign Up) และ ล็อกอิน (Login)
* **Access Control:** จำกัดสิทธิ์ให้เฉพาะผู้ใช้งานที่ผ่านการล็อกอินแล้วเท่านั้น ที่จะสามารถเข้าถึงและจัดการข้อมูลได้
